ATAK Integration Live

IonShield ships a live KML network link, a self-contained DDIL pack, and a Cursor-on-Target (CoT) feed — all consumable by any ATAK, WinTAK, or iTAK client without a custom plugin.

↓ Download Network Link KML ↓ Offline Pack (DDIL) CoT Feed

1. Live Overlay (Recommended)

Drop the network-link KML into ATAK once. ATAK will auto-refresh IonShield's risk zones every 5 minutes — no plugin required, durable across data-schema changes.

  1. Click Download Network Link KML above on the device.
  2. In ATAK, open Settings → Network Connections → Import Bundle and pick the file.
  3. The IonShield layer appears under Overlays. Toggle visibility from the overlay menu.
  4. Tap any colored region for current Kp / GPS / HF / SATCOM values.

2. Offline Pack (DDIL)

For Disconnected/Degraded/Intermittent/Limited operations: download a single self-contained KMZ pre-mission. Includes current risk + 24h Kp forecast. Refresh by re-downloading when comms return.

  1. On a connected device, fetch /atak/offline-pack.kmz.
  2. Transfer to the field device (USB / sneaker-net).
  3. In ATAK, Import → KML/KMZ file. Layer appears as a static overlay.

3. CoT Feed

For TAK Server pull: GET /overlay/ionshield.cot on a schedule. Each monitored location is a CoT event with stable UID (IONSHIELD-<id>) and ARGB color matching risk severity. For TAK Server push, set COT_SERVER_HOST in IonShield's env and we'll push events directly over TCP/8087.
