Shipping today
Auditable models in production
Two ML systems run live in the public deployment:
  • Event classifier — a multinomial logistic-regression model with softmax over 5 classes (BACKGROUND, GEOMAG_MAIN, SEP_EVENT, FLARE_M, FLARE_X). Trained on real NOAA data with synthetic class balancing for rare events.
  • 24-hour Kp forecaster — multi-horizon ridge regression predicting Kp at +1 / +3 / +6 / +12 / +24 h from a short lookback window of solar-wind drivers. Closed-form trained on the NASA OMNI archive plus live snapshots.
  • Champion / challenger A/B — every new model registers as a shadow challenger. Auto-promotes when it beats the live champion against operator feedback.
Pure-Python · readable JSON weights · auto-pilot retrain · drift metrics · per-tenant audit log
Near-term · 1–3 months
Specialised + customer-aware models
Three near-term lifts that do not require new science, just engineering and data:
  • Per-region heads — one specialised forecaster per geomagnetic-latitude band so polar regions and the equatorial anomaly get the precision they need.
  • Calibrated probabilities — Platt or isotonic scaling on top of the classifier so a "70% blackout probability" is empirically 70%, not raw softmax noise.
  • Customer feedback ground-truth — wire the customer's own GPS C/N0 logs, RTK fix-time, or HF SNR into the feedback store so the model is scored against the operator's actual experience.
Same auditable foundation · per-tenant model versions · explicit roadmap, not fantasy
Direction · 6–12 months
Learning from customer-specific environments
The long-term arc — stated as direction, not commitment:
  • Customer-tuned model versions — each customer's deployment carries its own promoted model, trained on its own environment and asset performance.
  • Mission-history-informed recommendations — accumulated mission and simulation data tells the system which decisions worked under which conditions; future recommendations get sharper.
  • Sequence / state-aware forecasting — predict not just Kp but storm phase (initial / main / recovery), so the recommendation can say "wait 90 min for recovery" instead of "monitor."
  • Decision-support that improves over time — the long-term arc is autonomy-friendly recommendation: IonShield suggests, the operator (or the system, if cleared) acts.
Stated as direction · technically reachable from today's foundation · no overclaiming
How customer learning works

A feedback loop, not a black box

The mechanism by which IonShield gets better for each customer over time. It's transparent, and it doesn't require giving up control of the customer's data.

01 · INGEST
Live + customer signals
Public NOAA + NASA feeds combine with the customer's own ground-truth (GPS C/N0, RTK fix-time, HF SNR, mission outcomes) inside the customer's own environment.
02 · PREDICT
Champion model serves
The current champion model issues predictions on every operator request. Each prediction is logged with its inputs and the operator-visible verdict.
03 · COMPARE
Ground-truth feedback
Operator outcomes (was the link actually viable? did the RTK actually hold?) come back as labels. Drift metrics surface when the model is no longer tracking reality.
04 · RETRAIN
Challenger spawns
A new model trains on the updated dataset, registers as a shadow challenger, and runs alongside the champion on real traffic — no production risk while we evaluate.
05 · PROMOTE
Auto-promote when better
When the challenger beats the champion across a sample window with margin, the system auto-swaps. Promotion is logged; rollback is one API call.
06 · COMPOUND
Customer-specific advantage
Over months, each customer's model converges on their own operating environment, asset mix, and risk tolerance. The same foundation; the customer's own outcome history.

Why this approach

Auditable wins over opaque

For defense, regulated commercial, and any operator who has to answer "why did the system tell you to do that?" — auditable matters more than sophisticated.

Readable model weights
Pure-Python regressors with JSON weights you can print to a screen. Compliance reviewers can read the model.
Shadow-mode before promote
No model ships to production users without proving itself on real traffic alongside the champion first. Promotion is reversible.
Physics as the floor
When the ML disagrees with physics-grounded models, physics wins by default. The ML adds, doesn't replace.
Operator feedback in the loop
Operator outcomes are first-class training signal. The model gets better at the operator's job, not at a benchmark.
