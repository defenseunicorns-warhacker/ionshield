Palantir Foundry Integration Live

IonShield writes Parquet to four Foundry datasets every 5 minutes and ships a ready-to-import ontology + Workshop layout so a Foundry-native customer can stand up a tactical space-weather app in their tenant in under an hour.

1. Datasets (Live)

Each dataset receives a Parquet append every 5 minutes. Schema is embedded in the file — the Foundry preview, SQL console, and Analyze Data view work without any “Apply Schema” click.

space_weather_raw

5-min NOAA SWPC + GloTEC merged feed; ground truth.

location_risk

Per-region (324 cells × time) impact metrics.

events

Detected storm onsets, SSCs, X-flares; one row per state transition.

impact

Long-form (region × time × system × metric) for time-series analytics.

2. Ontology Objects

Four Object Types map 1:1 to the datasets. Import them into Ontology Manager via JSON; the relations (Region ← StormEvent, Region ← ImpactRow) light up the Object Explorer and Workshop bindings.

  1. GET /api/v3/foundry/ontology — copy the JSON.
  2. In Foundry, open Ontology Manager → Import Object Types.
  3. Paste; map each backingDataset.envVar to the actual dataset RID in your tenant. (RIDs are listed in the Render dashboard's IonShield env vars.)
  4. Save. Object Explorer now shows IonShieldRegion and friends.

3. Workshop App

The pack ships a layout descriptor for a three-tab Workshop module: Live Globe, Storm History, Region Drill-in. Treat it as a spec — the Workshop UI is the source of truth for actual configuration.

  1. In Foundry, open Workshop → New Module.
  2. Use the layout from /api/v3/foundry/pack (see workshop_layout) as the build spec.
  3. Bind each widget's objectType to the matching Object Type imported above.

4. Sample SQL

Five queries ready to paste into Foundry's SQL console — last-hour Kp envelope, top 10 worst-impacted regions, storm history, GPS L1 outages, latest classifier version. GET /api/v3/foundry/sql for the full set.

5. Operational Notes

  • SNAPSHOT-on-deploy: first push after each deploy uses Foundry's SNAPSHOT transaction type to clean any legacy file format. Subsequent pushes APPEND.
  • Idempotent: writes are tagged with a wall-clock-millis filename, so replays don't double-count rows.
  • Failure-safe: a Foundry outage can never take down IonShield — sync errors are logged and swallowed.
  • Reset: POST /api/v3/foundry/reset forces the next push to be a SNAPSHOT.
