Features

Every capability your operators need
before mission launch

A single REST API and 3D dashboard covering ionospheric effects on GPS, HF, SATCOM, and radar — with typed go/no-go decisions and explainable confidence.

Core Capabilities

Route-specific. Platform-aware.
Confidence-tagged.

🌐
3D Route Risk Assessment
Click waypoints on a live Cesium 3D globe. Each route leg is colored by the worst ionospheric risk of its endpoints — NOMINAL (green) through SEVERE (red). Per-waypoint GPS error estimates and HF viability flags appear in the side panel and route profile bar.
POST /api/v2/route-decision
📡
HF / SATCOM Comms Decision
Point-to-point analysis for any lat/lon pair. Considers solar flux, IMF Bz polarity, Kp index, proton flux, and polar cap absorption. Returns a typed verdict: USE_PRIMARY_HF, SWITCH_TO_SATCOM, SWITCH_TO_UHF, or HF_NOT_VIABLE.
GET /api/v2/comms-decision
🔭
72-Hour Kp Forecast Timeline
Scrub forward in time with the forecast slider at the bottom of the globe. The ionosphere TEC overlay, auroral oval boundaries, and polar cap regions update in real-time to reflect forecast conditions. Essential for planning missions hours or days ahead.
GET /api/forecast
🎯
Explainable Confidence Scoring
Every decision carries a 0–1 confidence score with per-driver breakdown: data freshness, Bz variability, feed completeness, and stale-data penalty. Stale observations (>10 min) trigger a visible banner — never silent degradation of reliability.
HIGH · MEDIUM · LOW · VERY_LOW
⏱
Deterministic Observation Replay
Every NOAA observation is archived. Replay any past decision with the exact data snapshot used — same input hash, same output. Critical for after-action review, calibration, and regulatory audit trails.
GET /api/v2/replay
🔒
Full Data Provenance
Every API response includes a SHA-256 input hash, model version, compute timestamp, the exact observation and forecast feeds consumed, and a list of unavailable feeds. Fully auditable for operational compliance reviews.
SHA-256 · Model v1.0.0
