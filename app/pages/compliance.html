Where we are and

Status reflects our current architecture readiness and planned milestones, not formal certifications. Consult your compliance officer.

🏛️
FedRAMP High (Planned)
Planned · 12–18 mo
IonShield's cloud-native architecture is designed to run on AWS GovCloud, the target environment for FedRAMP High authorization. Container separation, TLS termination, and audit logging are implemented. Formal 3PAO assessment is planned for pilot customers reaching the enterprise tier.
🔐
CMMC Level 2 (In Progress)
In Progress · 6 mo
IonShield's software development practices align with NIST SP 800-171 (110 practices), a prerequisite for CMMC Level 2. We maintain an SBOM, use signed container images, and enforce MFA on all development infrastructure. A formal CMMC assessment is targeted concurrent with first defense-contractor pilot.
📋
DFARS 252.204-7012
Architecture Ready
IonShield does not process, store, or transmit Covered Defense Information (CDI) in its base configuration. Operator-supplied waypoint data (lat/lon pairs) is not classified. For customers who need to input sensitive route data, we provide a self-hosted deployment option with no external data egress.
🚀
ATO / Authority to Operate
Planned · per customer
IonShield can supply artifacts required for a customer-side ATO package: system security plan (SSP), data flow diagrams, dependency list, SBOM, and penetration test results. Each defense customer's ATO is unique — we support the process but do not hold a government-issued ATO.
🔑
Access Control
Implemented
API key authentication (X-API-Key header), rate limiting (slowapi), CORS restriction, strict security headers (CSP, HSTS, X-Frame-Options), and HTTPS enforcement. API keys are stored hashed; no plaintext secrets in version control.
📦
Air-Gap / Disconnected Ops
Roadmap Q3 2026
Zarf/UDS bundle packaging for disconnected deployment. IonShield operates in offline replay mode when NOAA feeds are unavailable. All stale-data warnings are displayed prominently when operating on cached data. Bundle includes all container images and tile data for the dashboard.
Export Controls

EAR99 — No export license required

Export Control Classification

IonShield software is assessed as EAR99 under the Export Administration Regulations (EAR). It does not incorporate any items on the Commerce Control List (CCL), does not use restricted encryption beyond standard TLS (FIPS 140-2 compliant libraries), and does not contain classified technical data.

The underlying NOAA SWPC data consumed by IonShield is public domain (U.S. Government work, not subject to copyright). CesiumJS is Apache-2.0 licensed. The full dependency SBOM is available upon request.

Note: This assessment is our good-faith classification. Customers are responsible for their own export-compliance review, particularly for international deployments.

ITAR Status

IonShield is not subject to the International Traffic in Arms Regulations (ITAR). It does not constitute a "defense article" under the United States Munitions List (USML) and does not provide military-specific capability beyond what is available through public NOAA data.

Security Practices

Secure development lifecycle

🛡️
Supply Chain Security
SBOM generated on every release. Dependencies pinned with hash verification. GitHub Actions CI with SLSA provenance. No third-party analytics in the dashboard (only the API server; marketing page uses GA4 with IP anonymization).
🔒
Data Minimization
IonShield does not require operator account creation. Waypoint data (lat/lon pairs) is processed in-memory and not persisted beyond the request lifetime. NOAA observation archives contain no operator data — only public NOAA feeds.
🔍
Audit Trail
Every decision carries a SHA-256 input hash, model version, and compute timestamp. All API calls are logged with timestamp and client IP (no PII). Audit logs are available for export on request.
🌐
Network Security
HTTPS enforced (HSTS). Strict CSP headers. CORS restricted to configured origins. Rate limiting on all endpoints (slowapi). No geolocation calls. No third-party JavaScript loaded in the operational dashboard.

Privacy Policy

Data Collected

Dashboard (operational): No personal data is collected. API key values are stored in your browser's localStorage only. No cookies are set by the operational dashboard.

Contact form: If you submit the pilot inquiry form, your email and organization name are stored securely and used only to respond to your inquiry. We do not sell or share this data.
